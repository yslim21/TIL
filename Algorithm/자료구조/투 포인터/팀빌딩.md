# 팀 빌딩
[팀 빌딩](https://www.acmicpc.net/problem/22945)

## 개념
+ 배열 양 끝에 포인터 배치
  + 두 포인터 사이에 1명 이상의 개발자가 존재하면 계속 탐색 
  + 두 포인터 중 큰 값을 가르키는 포인터를 이동시키면 min(개발자 A의 능력치, 개발자 B의 능력치)는 그대로 거나 더 작아지므로 팀 능력치의 최대값을 구할 수 없음
  + 두 포인터 중 작은 값을 가르키는 포인터를 이동시키면서 최대값 구하자 

## 풀이
```java
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());
        
        int N = Integer.parseInt(st.nextToken());
        int arr[] = new int[N];
        
        st = new StringTokenizer(br.readLine());
        for(int i=0; i <N; i++){
            arr[i] = Integer.parseInt(st.nextToken());
        }
        
        int start = 0; 
        int end = N-1; 
        int max =0; 
        
        while(start+1 < end){
            max = Math.max(max,(end-start-1)*Math.min(arr[start],arr[end]));
            if(arr[start] > arr[end]){
                end--;
            }else{
                start++;
            }
        }
        
        System.out.println(max);
    }
}
```
